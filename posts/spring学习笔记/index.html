Spring学习笔记

Written by jiewen with ♥ on 2020-05-07 in spring

Spring简介

Spring是一个分层的全栈轻量级开源框架，以Ioc和AOP为内核，提供了展现层SpringMVC和业务层事务等众多的企业级应用技术，还能整合众多开源的第三方框架，已经成为使用最多的java EE企业级应用开源框架。

Spring的优势

  • 方便解耦，简化开发

  • AOP编程的支持

  • 声明式事务的支持

  • 方便程序测试

  • 方便集成各种优秀框架

  • 降低使用Java EE API的难度

Spring的核心结构

Web、Data Access、Core container、Test、AOP/Aspects

IoC

IoC是什么？

IoC(Inversion of Control)控制反转

IoC解决了什么问题？

IoC之前，组件间的协调关系是通过new关键字来实现两组件之间的依赖关系的，造成了两组件之间的互相耦合，

IoC实现是在运行期将组件间的某种依赖关系动态的注入组件中。

IoC的实现方式方式以及DI

  1. 依赖查找(Dependency Lookup)

    容器中的对象通过容器的API来查找所依赖的资源和协作对象，无法在容器外部使用

  2. 依赖注入(Dependency Injection)

    依赖注入就是将被依赖的资源注入到使用它的地方，注入方式分为 setter和构造函数注入

Spring采用依赖注入的方式实现IoC容器

Spring IoC循环依赖

map 作用
singletonObjects 用于存放完全初始化好的bean，从该缓存中取出的 bean 可以直接使用
earlySingletonObjects 存放早期bean 对象（尚未填充属性），用于解决循环依赖
singletonFactories 存放 bean 工厂对象，填充的属性是循环依赖的话调用该工厂方法（实际去返回beanFactory的earlySingletonObjects中的bean），用于解决循环依赖

如图和表格所示，在bean的生命周期中，spring通过三个map来处理循环依赖。就是将未初始化属性的bean作为被依赖的对象设置到其它bean中

Lazy-Init延迟加载

单例可以延迟加载，其它Scope的bean不能延迟加载

延迟加载有两种配置方式

  • 注解@Lazy
  • xml

延迟加载是通过创建代理对象来实现的

FactoryBean和BeanFactory

  • BeanFactory

    是Spring Ioc 的顶层接口，提供了Ioc容器最基本的接口

  • FactoryBean

    Spring通过反射来实现指定类实例化，如果Bean的实例化过程比较复杂，Spring提供了FactoryBean接口，可以通过实现该接口来定制实例化Bean的逻辑，一般用于第三方框架集成到Spring

PostBeanProcessor处理器

在Bean的生命周期中，PostBeanProcessor在bean属性初始化之前和之后分别调用了Before…和After..方法，是Spring提供的可以参与Bean创建的生命周期的一个接口，用户可以通过实现该接口来修改Bean。Spring框架中的AutowiredAnnotationBeanPostProcessor（属性自动装配）、ApplicationContextAwareProcessor（Aware注入)等都是它的实现

AOP

什么是AOP？

AOP：Aspect oriented Programming 面向切面编程

AOP是OOP的延续，OOP的三大特性：封装、继承、多态，AOP采用横向抽取机制，取代了OOP纵向继承体系避免重复性代码的方式。

切：指的是横切逻辑，原方法逻辑代码不动，只操作横切逻辑代码

面：横切逻辑影响很多个方法，一个方法的横切如同一个点，多个点构成面

AOP解决了什么问题？

原业务不变的情况下，增强横切逻辑代码，也避免了横切逻辑重复性代码，达到解耦合的目的

Spring AOP

相关概念

  • Aspect（切面）： Aspect 声明类似于 Java 中的类声明，在 Aspect 中会包含着一些 Pointcut 以及相应的 Advice。

  • Joint point（连接点）：表示在程序中明确定义的点，典型的包括方法调用，对类成员的访问以及异常处理程序块的执行等等，它自身还可以嵌套其它 joint point。

  • Pointcut（切点）：表示一组 joint point，这些 joint point 或是通过逻辑关系组合起来，或是通过通配、正则表达式等方式集中起来，它定义了相应的 Advice 将要发生的地方。

  • Advice（增强）：Advice 定义了在 Pointcut 里面定义的程序点具体要做的操作，它通过 before、after 和 around 来区别是在每个 joint point 之前、之后还是代替执行的代码。

  • Target（目标对象）：织入 Advice 的目标对象.。

  • Weaving（织入）：将 Aspect 和其他对象连接起来, 并创建 Adviced object 的过程

在Spring中，通过cglib的动态代理功能实现了AOP，也封装了包括AspectJ切点表达式等功能

Spring 事务

关键对象

  • PlatformTransactionManager

    事务管理器，是管理事务的操作的，它只包含三个方法。获取事务，回滚事务，提交事务

  • TransactionDefiition

    定义事务的类型，事务包含很多属性，是否可读，事务隔离级别,事务传播级别。通过事务的定义，我们根据定义获取特定的事务。

  • TransactionStatus

    代表一个事务运行的状态，事务管理器通过状态可以知道事务的状态信息，然后进行事务的控制。事务是否完成，是否是新的事务，是不是只能回滚等

编程式事务

​ 直接在代码中使用Spring提供的对象来实现事务功能

声明式事务

​ 使用@EnableTransactionManagement开启事务（同时Spring注册相关BeanDefinition）

​ 使用@Transactional来标记类或方法使用事务，并且可以设置事务的各种信息
