Spring Boot核心注解

启动类上的@SpringBootApplication注解，包含了三个注解：

@SpringBootApplication

  • @SpringBootConfiguration

    • Configuration //配置类

  • @EnableAutoConfiguration

    • @AutoConfigurationPackage

      • @Import(AutoConfigurationPackages.Registrar.class) //把用到的bean导入到容器

    • @Import({AutoConfigurationImportSelector.class})

  • @ComponentScan //指定basePackage，容器启动会扫描指定包下的类自动注册到容器

@Configuration与@Component的区别

  • @Configuration

    1. @Configuration是ConfigurationClassPostBeanProcessor通过cglib的enhance方法来生成动态代理的类(代理类继承原来的类，自动实现了BeanFactoryAware接口, 并新增一个beanFactory字段)，并设置方法的callback拦截器，当代理类调用时会执行callback拦截器的拦截方法，对bean实例化后注册到BeanFactory

    2. 其中所有包含@Bean的方法都会被拦截，同一类中的其它@Bean方法调用前会检查是否已经实例化，如果已经实例化就通过 beanFactory.getBean 获取，如果没有，就调用原方法。例如，调用UserInfo实例化时会先获取，如果没有就调用原方法生成bean，这里的bean是同一个

      @Configuration
public class MyConfiguration {
    @Bean
    public AccountInfo accountInfo(){
       return new AccountInfo(100);
    }
       
    @Bean
    public UserInfo userInfo(){ 
        User user = new User(); 
        user.account = accountInfo();
        return user;
    }
}

  • @Component没有通过cglib代理bean的创建方法，类似方式就会是两个不同的bean

@Import

注解可以类导入到 IoC容器中。该注解有三种使用方式：

  • 导入普通类
    • 导入指定类
  • 通过ImportSelector 方式导入
  • 导入的类实现ImportSelector接口，返回需要导入类的全限定名列表
  • 通过 ImportBeanDefinitionRegistrar 方式导入
    • 导入的类实现ImportBeanDefinitionRegistrar 接口，自定义注册过程

@Import(AutoConfigurationPackages.Registrar.class)

将AutoConfigurationPackages.Registrar导入到容器，其中，将当前类的package作为参数保存在AutoConfiguration.BasePackages 静态内部类中保存，如果有多个，则合并添加，所以，在其它module中，也可以扫描标记类的包。代码中可以通过AutoConfigurationPackages.get(beanFactory)的方式获取已经注册进去的basePackages

@Import({AutoConfigurationImportSelector.class})

实现了ImportSelector接口，selectImports 方法中通过获取本地资源spring.factories，获得设定好的类列表，并按照spring-autoconfigure-metadata.properties配置好的加载方式，来实现自动加载

Spring boot 配置文件

yaml和properties

相比 properties 配置文件，

YAML 配置简洁有序并且支持数组类型

YAML 还有一个缺点，就是不支持 @PropertySource 注解导入自定义的 YAML 配置

Spring还支持xml格式的配置

bootstrap.properties 和 application.properties 有何区别 ?

  • bootstrap：boostrap 由父 ApplicationContext 加载的，比 applicaton 优先加载，配置在应用程序上下文的引导阶段生效。一般来说我们在 Spring Cloud Config 或者 Nacos 中会用到它。且 boostrap 里面的属性不能被覆盖
  • application： 由ApplicatonContext 加载，用于 spring boot 项目的自动化配置。
